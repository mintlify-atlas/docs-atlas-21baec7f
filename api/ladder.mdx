---
title: Get Ladder
api: GET /api/ladder
description: Retrieves the complete ranked ladder with all friends sorted by Solo/Duo rank
---

This endpoint returns the full ladder of all tracked friends, sorted by Solo/Duo rank from highest to lowest. The data is cached and refreshed on a configurable interval.

## Request

<CodeGroup>
```bash cURL
curl http://localhost:3001/api/ladder
```

```javascript JavaScript
const response = await fetch('http://localhost:3001/api/ladder');
const ladder = await response.json();
```

```python Python
import requests

response = requests.get('http://localhost:3001/api/ladder')
ladder = response.json()
```
</CodeGroup>

  Array of player objects sorted by Solo/Duo rank (highest to lowest)

    <ResponseField name="tagLine" type="string">

    <ResponseField name="soloq" type="object | null">

        <ResponseField name="rank" type="string">

        </ResponseField>
        
        <ResponseField name="wins" type="number">
          Wins this season

        <ResponseField name="losses" type="number">

        </ResponseField>

    </ResponseField>
    
    <ResponseField name="flex" type="object | null">
      Ranked Flex queue data (same structure as soloq, queueType: "RANKED_FLEX_SR")
    </ResponseField>
    
    <ResponseField name="topChampions" type="string[]">
      Array of up to 3 most played champion names from recent 20 ranked matches
    </ResponseField>
    
    <ResponseField name="mainRole" type="string | null">
      Most played role: TOP, JUNGLE, MIDDLE, BOTTOM, or UTILITY
    </ResponseField>
    
    <ResponseField name="emote" type="string | null">

    </ResponseField>
    
    <ResponseField name="error" type="string | null">

    </ResponseField>

      True if data is from cache due to rate limiting

  </Expandable>
</ResponseField>

<ResponseField name="cachedAt" type="string | null" required>
  ISO 8601 timestamp of last successful cache refresh
</ResponseField>

<ResponseField name="cacheTtlMs" type="number" required>
  Cache TTL in milliseconds (default: 60000)
</ResponseField>

<ResponseField name="stale" type="boolean" required>
  True if the last refresh encountered an error
</ResponseField>

<ResponseField name="lastError" type="string | null" required>
  Error message from last refresh attempt, or null if successful
</ResponseField>

<ResponseField name="ddragonVersion" type="string" required>
  Current Data Dragon version (e.g., "14.24.1")
</ResponseField>

## Behavior

<Info>
**Incremental Refresh**: The API doesn't refresh all players on every interval. It refreshes a configurable batch (default: 2 players per cycle) determined by `FRIENDS_PER_REFRESH` in `.env`.
</Info>

<Note>
**First Request**: If the cache is empty and no refresh is in progress, the endpoint triggers a full refresh before responding.
</Note>

<Warning>
**Rate Limiting**: When the Riot API rate limit is hit, the endpoint returns the last successfully cached data with `stale: true` and includes the error in `lastError`.
</Warning>

## Sorting

Players are sorted by Solo/Duo rank using this priority:

1. Tier (CHALLENGER > GRANDMASTER > MASTER > DIAMOND > EMERALD > PLATINUM > GOLD > SILVER > BRONZE > IRON)
2. Division (I > II > III > IV)
3. LP (higher > lower)

Players without Solo/Duo rank appear at the bottom.

## Use Cases

- Display a real-time ranked ladder on a web dashboard
- Track friends' rank progression over time
- Build leaderboards with custom visualizations
- Monitor when friends are climbing or falling in rank
