The status endpoints provide detailed information about the API's operational state, rate limiting, cache statistics, and daily LP gain/loss highlights.

## Get Status

<api>GET /api/status</api>

Retrieve comprehensive API status including rate limit info, cache stats, and daily highlights.

### Request

<CodeGroup>
```bash cURL
curl http://localhost:3001/api/status
```

```javascript JavaScript
const response = await fetch('http://localhost:3001/api/status');
const status = await response.json();

console.log(`Rate limited: ${status.riotRateLimited}`);
console.log(`Budget remaining: ${status.budgetRemaining1s}/18 (1s), ${status.budgetRemaining2min}/90 (2m)`);
```
</CodeGroup>

<ResponseField name="totalRequests" type="number" required>

<Info>

</Note>

## Force Refresh

<api>POST /api/force-refresh</api>

Manually trigger an immediate full refresh of all players in the ladder, bypassing the incremental refresh cycle.

### Request

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3001/api/force-refresh
```

```javascript JavaScript
const response = await fetch('http://localhost:3001/api/force-refresh', {
  method: 'POST'
});
const result = await response.json();

console.log(result.message);
```
</CodeGroup>

### Response

<ResponseField name="ok" type="boolean" required>
  True if refresh completed successfully
</ResponseField>

<ResponseField name="message" type="string" required>
  Description of what happened
</ResponseField>

<ResponseField name="playersCount" type="number" required>
  Number of players in the refreshed ladder
</ResponseField>

### Response Examples

<CodeGroup>
```json Success
{
  "ok": true,
  "message": "Refresh completado",
  "playersCount": 9
}
```

```json Already Refreshing
{
  "ok": true,
  "message": "Ya había un refresh en curso, completado",
  "playersCount": 9
}
```

```json Error - No API Key
{
  "error": "Sin RIOT_API_KEY"
}
```

```json Error - Refresh Failed
{
  "error": "Riot API rate limit exceeded"
}
```
</CodeGroup>

### Behavior

<Warning>
**Rate Limit Risk**: Forcing a full refresh fetches data for ALL friends, which can quickly consume your rate limit budget. Use sparingly.
</Warning>

<Info>
**Refresh Queue**: If a refresh is already in progress, this endpoint waits for it to complete and returns the result instead of starting a new one.
</Info>

<Note>
**Cursor Reset**: After a forced full refresh, the incremental refresh cursor resets to 0.
</Note>

## Rate Limiting Details

The API implements both proactive and reactive rate limiting:

### Proactive (Sliding Window)

- **18 requests / 1 second**
- **90 requests / 2 minutes**

The API automatically throttles requests to stay within these limits, with a 60ms safety buffer.

### Reactive (429 Responses)

When Riot returns a 429 status:

1. The API extracts the `Retry-After` header
2. Sets a hard block until that time
3. Returns cached data for subsequent requests
4. Marks responses as `stale: true`

<Info>
You can configure the fallback retry delay via `RATE_LIMIT_FALLBACK_MS` in `.env` (default: 60000ms).
</Info>

## Daily Highlights

Daily LP tracking starts fresh each day at midnight (server timezone) and tracks:

- **Best Solo/Duo gain**: Highest LP gained in Solo/Duo today
- **Best Flex gain**: Highest LP gained in Flex today
- **Worst Solo/Duo loss**: Most LP lost in Solo/Duo today
- **Best overall gain**: Highest combined LP gain today
- **Worst overall loss**: Most combined LP lost today

<Note>
LP deltas are calculated incrementally on each refresh, not just from start to end of day.
</Note>
