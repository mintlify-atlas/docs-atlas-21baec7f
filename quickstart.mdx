Get your custom League of Legends ladder up and running in minutes. This guide will walk you through installation, configuration, and launching your ladder tracker.

## Prerequisites

Before you begin, ensure you have:

- **Node.js 20+** and **npm 10+** installed
- A **Riot Games API key** (get one at [Riot Developer Portal](https://developer.riotgames.com/))

<Note>
  The Riot API personal development key has rate limits of 20 requests/second and 100 requests/2 minutes.
</Note>

## Installation

<Steps>
  <Step title="Clone the Repository">
    First, clone the repository to your local machine:
    
    ```bash
    git clone <your-repo-url>
    cd tullidos-soloq-ladder
    ```
  </Step>

  <Step title="Install Dependencies">
    Install dependencies for both the server and client:

    <CodeGroup>
    ```bash npm
    # Install server dependencies
    cd server
    npm install

    # Install client dependencies
    cd ../client
    npm install
    ```

    ```bash yarn
    # Install server dependencies
    cd server
    yarn install

    # Install client dependencies
    cd ../client
    yarn install
    ```

    ```bash pnpm
    # Install server dependencies
    cd server
    pnpm install

    # Install client dependencies
    cd ../client
    pnpm install
    ```
    </CodeGroup>
  </Step>

  <Step title="Configure Environment Variables">
    Create your environment configuration file:

    ```bash
    cd server
    cp .env.example .env
    ```

    Open `server/.env` and add your Riot API key:

    ```bash server/.env
    PORT=3001
    RIOT_API_KEY=RGAPI-your-api-key-here
    LADDER_CACHE_TTL_MS=60000
    MATCH_SYNC_TTL_MS=1800000
    RATE_LIMIT_FALLBACK_MS=60000
    FRIENDS_PER_REFRESH=2
    ```

    <Warning>
      Never commit your `.env` file to version control. Keep your API key secret!
    </Warning>

    ### Configuration Options

    - `PORT` - Server port (default: 3001)
    - `RIOT_API_KEY` - **Required** - Your Riot Games API key
    - `LADDER_CACHE_TTL_MS` - Cache refresh interval in milliseconds (default: 60000 = 1 minute)
    - `MATCH_SYNC_TTL_MS` - Match history sync interval (default: 1800000 = 30 minutes)
    - `RATE_LIMIT_FALLBACK_MS` - Fallback delay when rate limited (default: 60000)
    - `FRIENDS_PER_REFRESH` - Number of players to refresh per cycle (default: 2)
  </Step>

  <Step title="Add Players to Track">
    Edit `server/friends.json` to add the players you want to track:

    ```json server/friends.json
    [
      {
        "gameName": "Azpy",
        "tagLine": "1337",
        "mote": "Azpy"
      },
      {
        "gameName": "NEWJEANS HANNI",
        "tagLine": "1975"
      }
    ]
    ```

    <Info>
      - `gameName` and `tagLine` are required for each player
      - `mote` is an optional custom nickname/emoji to display
      - `puuid` will be automatically populated after the first refresh
    </Info>
  </Step>

  <Step title="Start the Application">
    Launch both the server and client in separate terminal windows:

    **Terminal 1 - Start the server:**

    <CodeGroup>
    ```bash npm
    cd server
    npm run start
    ```

    ```bash yarn
    cd server
    yarn start
    ```

    ```bash pnpm
    cd server
    pnpm start
    ```
    </CodeGroup>

    **Terminal 2 - Start the client:**

    <CodeGroup>
    ```bash npm
    cd client
    npm run dev
    ```

    ```bash yarn
    cd client
    yarn dev
    ```

    ```bash pnpm
    cd client
    pnpm dev
    ```
    </CodeGroup>

    <Tip>
      For development with auto-reload, use `npm run dev` instead of `npm run start` in the server directory.
    </Tip>
  </Step>

  <Step title="Access Your Ladder">
    Open your browser and navigate to:

    - **Frontend (Ladder UI):** [http://localhost:5173](http://localhost:5173)
    - **Backend API:** [http://localhost:3001](http://localhost:3001)

    Your ladder will automatically start fetching player data from the Riot API!
  </Step>
</Steps>

## Troubleshooting

### API Key Issues

If you see errors about missing `RIOT_API_KEY`:

1. Verify the key is correctly set in `server/.env`
2. Make sure there are no extra spaces or quotes around the key
3. Restart the server after updating the `.env` file

### Rate Limit Errors

The application automatically handles Riot API rate limits, but if you're hitting limits frequently:

- Increase `LADDER_CACHE_TTL_MS` to reduce refresh frequency
- Reduce `FRIENDS_PER_REFRESH` to update fewer players per cycle
- Check `/api/status` endpoint to monitor your API usage

### Empty Ladder

If your ladder appears empty:

1. Check that `server/friends.json` contains valid player names and tags
2. Verify your API key has the correct permissions
3. Check the server console for error messages
4. Visit `http://localhost:3001/api/status` to see the refresh status
